`AIRecipeFinder` is a **Django + Tailwind CSS** web application that generates recipes using **Groq Llama-3** and fetches cooking tutorial videos from **YouTube**.

A key feature of this project is the inclusion of **mock fallback data**. This ensures the UI remains fully functional for development and testing, even without active API keys.

## ğŸš€ Getting Started (Local Development)

Follow these steps to get the project running on your local machine.

### 1. Prerequisites

* [Python](https://www.python.org/downloads/) (3.11+)
* [Node.js and npm](https://nodejs.org/en) (for Tailwind CSS)
* Git

### 2. Clone the Repository

```bash
git clone [https://github.com/YOUR_USERNAME/AIRecipeFinder.git](https://github.com/YOUR_USERNAME/AIRecipeFinder.git)
cd AIRecipeFinder
```

### 3. Set Up Environment Variables

Create a `.env` file in the project root by copying the example:

```bash
cp .env.example .env
```

Now, open the `.env` file and add your credentials.

```ini
# .env

# Django settings
# Generate a new key from [https://djecrety.com/](https://djecrety.com/)
SECRET_KEY="your_django_secret_key"
DEBUG=True

# API Keys (Optional: App will use mock data if left blank)
GROQ_API_KEY=""
YOUTUBE_API_KEY=""
```

### 4. Set Up the Backend (Django)

```bash
# Create and activate a virtual environment
python -m venv venv
source venv/bin/activate  # (on Windows, use `venv\Scripts\activate`)

# Install Python dependencies
pip install -r requirements.txt

# Apply database migrations
python manage.py migrate

# Create a superuser (for admin access)
python manage.py createsuperuser
```

### 5. Set Up the Frontend (Tailwind CSS)

You'll need a separate terminal for this.

```bash
# Install Node.js dependencies
npm install

# Run the Tailwind CSS build process in "watch" mode
# This will automatically rebuild your CSS on file changes.
npm run tailwind:watch
```

### 6. Run the Application

With the Tailwind watch process running in one terminal, open a second terminal to run the Django server.

```bash
# (Activate your virtual environment first if it's not already)
source venv/bin/activate

# Run the Django development server
python manage.py runserver
```

You can now access the application at **http://127.0.0.1:8000/**.

## â˜ï¸ Deployment Notes

* Before deploying to production, ensure you set **`DEBUG=False`** in your environment variables.
* You will need to run `python manage.py collectstatic` to gather all static files (including the compiled Tailwind CSS) into the `staticfiles` directory.
* Ensure all necessary environment variables (like `SECRET_KEY`, `GROQ_API_KEY`, etc.) are set in your production environment.
